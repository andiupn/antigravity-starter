## 💡 Проблема: почему традиционные рабочие пространства не справляются с искусственным интеллектом
Агенты ИИ (Курсор, Близнецы, Клод) феноменально способны. Но при попадании в стандартные, захламленные каталоги они теряются. Они читают ненужные папки, тратят ваш бюджет токенов, галлюцинируют и размещают файлы не в тех каталогах, потому что нет четких границ.

---

## ⚡ Решение: три столпа премиальной системы искусственного интеллекта

### 1. 🛰️ Идеальное видение для ИИ (AI-First Layout)
Мы разработали каждую папку, конфигурацию и правило `.gitignore` так, чтобы они служили дорожными картами для вашего ИИ. Рабочее пространство чрезвычайно чистое, что позволяет Cursor или Gemini понять всю вашу архитектуру **Next.js 16 + React 19 + Tailwind v4** менее чем за 3 секунды. Ноль потраченных жетонов, максимальная точность.

### 2. 🤖 Специализированная команда, а не обычный чат-бот
Вы не разговариваете с пустым ящиком ИИ. Вы получаете команду из двух предварительно настроенных, узкоспециализированных и автономных ИИ-агентов:
- **`@code-reviewer`** — Ваша скрупулезная и строгая команда контроля качества, которая проверяет качество кода, его структуру и структуру перед фиксацией.
- **`@research-assistant`** — Высокоскоростной и экономичный исследовательский агент, предназначенный для безопасного изучения новых документов и библиотек API.

### 3. ⚡ Навыки работы в один клик (рабочие процессы)
Выполняйте сложные рабочие процессы разработки с помощью простых команд. Оснащен 8 многократно используемыми рабочими процессами автоматизации (`/skills`), которые превращают ручное создание шаблонов, проверку среды и написание тестов в задачу, занимающую 1 секунду.

## 📂 План рабочего пространства

```
your-workspace/
  .gemini/             # AI agent configs, automation skills, and wiki
  docs/                # Pristine research and implementation plans
  src/                 # Main directory for your application source code
  .env.example         # Template for environment variables
  .gitignore           # Standard Git ignore file
  .mcp.json            # Model Context Protocol (MCP) configuration
  GEMINI.md            # AI coding rules & workspace guidelines
  LICENSE              # Project license (MIT License)
```

## 🚀 Начните за 3 шага

### 1. Скопируйте конфигурации:

```bash
cp .env.example .env
```

*(Вставьте свой `GITHUB_TOKEN`, чтобы включить интеграцию с репозиторием).*

### 2. Настройка разрешений ИИ:

```bash
cp .gemini/settings.local.example.json .gemini/settings.local.json
```

*(Раскомментируйте и установите `permissions` в соответствии с вашей ОС).*

### 3. Начните кодирование:
Откройте это рабочее пространство в своем любимом редакторе с поддержкой искусственного интеллекта и оцените скорость.
