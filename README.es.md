## 💡 El problema: por qué los espacios de trabajo tradicionales fallan en la IA
Los agentes de IA (Cursor, Gemini, Claude) son fenomenalmente capaces. Pero cuando se los coloca en directorios estándar y abarrotados, se pierden. Leen carpetas irrelevantes, gastan su presupuesto simbólico, alucinan y colocan archivos en directorios equivocados porque no hay límites claros.

---

## ⚡ La solución: tres pilares del andamiaje de IA premium

### 1. 🛰️ Visión perfecta para la IA (AI-First Layout)
Diseñamos cada carpeta, configuración y regla `.gitignore` para que actúen como hojas de ruta para su IA. El espacio de trabajo es extremadamente limpio, lo que permite a Cursor o Gemini comprender toda su arquitectura **Next.js 16 + React 19 + Tailwind v4** en menos de 3 segundos. Cero tokens desperdiciados, máxima precisión.

### 2. 🤖 Equipo especializado, no un chatbot genérico
No se habla con un cuadro de IA en blanco. Obtienes un equipo de 2 agentes de IA autónomos, altamente especializados y preconfigurados:
- **`@code-reviewer`**: su riguroso y estricto equipo de control de calidad que revisa la calidad del código, las pelusas y la estructura antes de realizar las confirmaciones.
- **`@research-assistant`**: un agente de investigación rentable y de alta velocidad diseñado para explorar nuevas bibliotecas y documentos API de forma segura.

### 3. ⚡ Habilidades con un clic (flujos de trabajo)
Ejecute flujos de trabajo de desarrollo complejos con comandos simples. Equipado con 8 flujos de trabajo de automatización reutilizables (`/skills`) que convierten el andamiaje manual, la verificación del entorno y la redacción de pruebas en una tarea de 1 segundo.

## 📂 Plano del espacio de trabajo

```
your-workspace/
  .gemini/             # AI agent configs, automation skills, and wiki
  docs/                # Pristine research and implementation plans
  src/                 # Main directory for your application source code
  .env.example         # Template for environment variables
  .gitignore           # Standard Git ignore file
  .mcp.json            # Model Context Protocol (MCP) configuration
  GEMINI.md            # AI coding rules & workspace guidelines
  LICENSE              # Project license (MIT License)
```

---

## 🚀 Comience en 3 pasos

### 1. Copiar configuraciones:

```bash
cp .env.example .env
```

*(Inserte su `GITHUB_TOKEN` para habilitar las integraciones del repositorio).*

### 2. Configurar permisos de IA:

```bash
cp .gemini/settings.local.example.json .gemini/settings.local.json
```

*(Descomenta y configura `permissions` según tu sistema operativo).*

### 3. Comience a codificar:
Abra este espacio de trabajo en su editor compatible con IA favorito y experimente la velocidad.
